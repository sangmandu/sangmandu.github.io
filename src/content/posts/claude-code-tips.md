Claude Code는 단순한 코드 생성기가 아니다. 진짜 가치는 대화의 맥락을 유지하면서 복잡한 작업을 자율적으로 수행하는 데 있다.

많은 사람들이 Claude Code를 처음 쓸 때 가장 흔히 하는 실수는, 하나하나 지시하려 하는 것이다. "이 파일 열어", "이 함수 수정해", "테스트 돌려" — 이렇게 마이크로매니징하면 사람이 직접 하는 것보다 오히려 느려진다.

## 의도를 전달하라, 명령을 내리지 마라

핵심은 *무엇을* 원하는지 명확히 하고, *어떻게*는 맡기는 것이다. "로그인 페이지에 소셜 로그인 추가해줘"가 좋은 프롬프트다.

> 좋은 프롬프트는 결과를 설명하고, 나쁜 프롬프트는 과정을 지시한다.

## 컨텍스트가 전부다

Claude Code는 CLAUDE.md, 프로젝트 구조, git history를 읽는다. 이 컨텍스트가 풍부할수록 결과가 좋아진다. README를 잘 써놓으면 프롬프트를 덜 써도 된다.

## 스킬과 훅을 활용하라

반복되는 워크플로우는 스킬로 만들고, 자동화가 필요한 부분은 훅으로 해결한다. 한 번 세팅해두면 매번 같은 설명을 반복하지 않아도 된다.
